Multiple Inheritance : Concept and Implementation in C++

And yes we do have solution for DIAMOND PROBLEM.

Posted by Stone on September 29, 2017

INHERITANCE is the concept to reuse the older functionality for newer class. All the implemented properties are inherited in the newer class. So at board level it prevent us from reinventing the whole wheel again. C++ supports all types of inheritance, which includes:

  • Simple inheritance
  • Multi-level inheritance
  • Multiple inheritance
  • Hybrid inheritance

In this article our main focus will be Multiple-Inheritance.Multiple inheritance is a inheritance in which we can have N base classes and a single derived class inheriting from more than one BASE CLASS

Fig 1. Multiple-Inheritance Block Diagram

Many of OOPS languages don't supports MULTIPLE INHERITANCE directly , as it has its own issues which we will discuss in later shortly.

Let us take a real world example of multiple inheritance, which will make it more clear for us.

Today most of us uses smart phones, which has many functionality but for our examples lets consider we need a smart phone with two basic functionality Phone(Calling and SMS) and Camera (Take pictures). Both are separate entities and have their own separate implementation of their functionalities. A SmartPhone developer instead of developing from zero, will use the these already existing implementation of the functionalities. Below is the diagram representing :

Fig 2. Class Diagram from above example

Basic code syntax of multiple inheritance

class Phone { } ;
class Camera { } ;
class SmartPhone : public Phone , public Camera ;

Memory structure , how multiple inheritance is plotted in memory :

+-----------+
|...Phone...+
+-----------+
|...Camera..+
+-----------+
|Smart_Phone+
+-----------+

Issues with multiple inheritance :

  • Ambiguity

    It is compile time error in C++, which says that in the MULTIPLE-INHERITANCE design, either of base classes have same function and calling from derived class confuses complier which one to call.

    Continuing with the smart-phone example , lets say we have sleep functionality in both Classes(Phone and Camera), and we have a smartPhone. From smartPhone we called sleep functionality. Here whats the problem arises ,which functionality is to be called (Phones or Camera). Hence we have ambiguous condition here.

    It will be more clear with an example

    Fig 3.Ambiguity problem.

    As smartPhone has 2 definition of sleep, it will be not able to differentiate among them and compilation error of " ambiguous member " is shown

  • DIAMOND PROBLEM

    Fig 3.Diamond Problem

    From the above diagram , we can see that the DerivedClass2 has two copies of Base, which in memory looks like :

    Fig 3.Memory plot of diamond problem.
    This is the diamond problem.

None feature in any programming language is neither too good that we can use it at every problem nor too bad that it has no use case of it. Every feature has it well defined range. we cannot avoid multiple-inheritance completely, it is a powerful tool for its range. Yes, when multiple inheritance is being used, be aware of the pit fall like diamond problem and ambiguity.
